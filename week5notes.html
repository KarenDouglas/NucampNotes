Week 5 Notes

Review: Function Declarations

Functions

  • Functions are a piece of code that can be used over and over again
  • There are multiple ways to create a function in js. Function declaration is the oldest way

    • function functionName( parameters) {
    // the code to be executed
    }

  • In order to run the function code, you have to call the function.To call a function:

    • uniqueFunctionName(argument);

  • These functions can be called over and over again

The Parameter List and Arguments

  • you can pass literal values or variables into a function; these are called Arguments

    • function uniqueFunctionName( parameter 1,parameter 2, parameter 3) {
    // function body-the statements that are executed when the function is called
    }

    uniqueFunctionName(ARGUMENTS);

Multiple Parameters

  • multiple parameters can pass thru a function. The arguments simply have to link to each parameter

Function Invocation

  • The code inside the function will execute when something invokes or calls the function:
    • When event occurs
    • when it is called from JS code
    • Automatically (self-invoked)

The Return Statment

  • regardless to what happens outside a function, a function returns a value
  • if you don't set a function to be returned, its value will be undefined
  • When JavaScript reaches a return statement, the function will stop executing.
  • Return statements specifies a value to be passed thru the function

Function Hoisting

  • All functions written with the function declaration syntax will be pulled to the top of its scope before execution. aka you will be able to call that function even before you reach it

Review: Events

Onevent Handlers

  • this is the old fashion way
  • there are two ways to write an onevent Handler
    • the first way is to write a html attribute, the event can be called in js

      • <elementName oneventHandlerName="event('js')" > text < elementName>

    • writing onEvent handlers in this way is not recommended
    • The other way to write onevent handlers is to use querySelectors in JS:

      • document. querySelector('elementName').event= functionName;

      //followed by the function or functions that defines the functionName

Event Listeners

  • You can add an event listener to any download

    • selector.addEventListener('event', functionName);

  • the advantage to event listeners is that you can attach more than one for the same event to one node
  • you can also remove events in JS code like this:

    • selector.removeEventListener('event', functionName)

JS: Function Expressions

Function Expressions

  • very similar to a function declaration, a function expression has almost the same syntax
  • the main diff between them is that a function expression doesn't require a function name( anonymous function) ex:

    • const constName =function (a,b){
    return a+b;
    };
    console.log(constName)

  • another diff is that expressions can't be hoisted
  • if you want to refer to the current function inside the function body, you need to creat a named expression.(this name is then local only to the function body scope, which is rare)

Arrow Functions

Arrow Functions

  • add to js in ES6
  • the two main advantages to arrow functions: 1. they are more concise. 2. they dont have their own "this" binding

    • variable variableName = (a,b) => a+b;
    variableName(argument);

  • you can omit the parentheses if there is only one variable
  • you can only omit the curly braces if there is only one statement in the function

    • variable variableName = (a,b) => {
    variable variableName2 =a + b;
    return c;
    }

  • arrow functions can not be hoisted

Var, Let, Const

The diff between let and const

  • let can be reassigned and const can not
  • const can not be initialized without declaration. let can be initialized with out being declared
  • its recommended to used consts in most cases unless you know you will be reassigning the variable.

Var vs. Let and Const

  • var can be redeclared, let and const can not
  • var has function scope; let and const have block scope

Global, function, and block scope

  • the scope defines where a variable is accessible
  • global scope: a variable declared outside any code block, regardless of if yo use let, var, or const
    • it's best to limit declaring variables globally, as it is harder to keep track of them as the js script gets longer. too many variable in global scope is known as "poluting the global scope"
  • var has funtion scope. If you declare a variable using var inside a function, the variable only exists inside that function and cannot be accessed from outside of that function. It's only created when the function is called, then deleted when the function call is finished.
  • In general, child scopes in JavaScript will inherit the variables of their parent scope, regardless of if the variables were declared with var, let, or const. This is a one-way relationship - parents do not inherit the variables of their child scopes.
  • let and const have block scope, which means that they exist only within the code block they are declared. these code blocks can be any block; function, for,while loop, switch, if etc.
    • for example: if you set a var inside of an If block, it will exist outside of the If block. If you set a let or const inside of an if block, it only exists inside of that if block
  • tip: be sure to declare you variable before using them
