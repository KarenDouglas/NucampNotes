Week 2 Notes

Bootstrap Components

  • Sets of classes that help you create many common parts of a user interface
  • The source code for these components can b e based in purely Css or a combo of Css and JavaScript

Color and Spacing Utility Classes

  • These are the limited amount of colors available in Boostrap.
  • Boostrap includes a wide range of shorthand responsive margin and padding utility classes to modify an element's appearance
    • The classes are named using the format {property}{sides}-{size} for xs and {property}{sides}-{Breakpoint}-{Size} for sm,md,lg,and xl.

    • Overview: Navigation

    • Navbar: A bar with links to main pages on a websites. Boostraps navbar can easily become responsive, change the bar to a toggler on smaller view-ports
    Breadcrumbs

    There are 3 types of Breadcrumbs

    • Location: static, shows where the page is located in the website hierarchy
    • Attribute: give info that categorizes the current page based on attributes chosen by the user.
    • Path: Show users the steps they took to get to the current page- aka "history trail"- least often used type of breadcrumb
    Do's and Don'ts of Navigation Design
    • Use user-friendly, simple terms
    • Be consistent- provide the same navigation throughout your website-
    • Ensure that clicking the name/logo of your website in the header should take users back to the home page
    • Don't clutter the main navigation with too many options
    • Don't use vague terms
    • Don't make it hard for users to find out where they are in the hierarchy of your website
    How-to Navigation
    • Navbars require a wrapping .navbar with .navbar-expand{-sm|-md|-lg|-xl} for responsive collapsing and color scheme classes.

    # in link

    The "#" in the navigation links will take the user to the current page they are on.

    Icons Fonts, Buttons and Forms

    Icon Fronts
    • web dev once used image files for icons. It was even necessary to create an entirely new image file to change the color, and other styling for and icon.
    • modern web dev use icons fonts or SVG icons
    Buttons
    • Use the buttons in bootstrap with bootstrap social icons to give your icons a nicer touch
    • Use Bootstraps custom button styles for actions in forms, dialogs, and more with support for multiple zizes,states and more
    • the .btn classes are designed to be used with the "button" element.However you can also use these classes on "a" and "input" elements
      • when using button classes on "a" elements that are used to trigger in-page functionality(like collapsoing content), rather than linking to new pages or sections within the current page, these links should be given a role:"button" to appropriately convey their purpose to assistive tech such ass screen readers
    Forms
    1. Use {form}{form/} tag
    2. {input/} determine input type,name, placeholder and I.D.
    3. {label}{label/} use label to connect text with its related input. Identify "for". Labels are also useful for screen readers
      • Labels for input: type:radio can have the same id for more than one of its inputs as it may be another option in one query
    Form Layout using Grid and Nested Grid
    • more complex forms can be built using grid classes. Use these for form layouts that require mulitple columns,varied widths, and additional alignment options
    • you can create more rows and columns nested inside of a column
    Bootstrap Form Component Classes
    • Boostrap Form-group class (.form-group): helps with default alignment for elements
    • Form control class (.form-control):applies default styling that makes the input fields look better, such as padding
    • Column form label (.col-form-label):causes label to align vertically with its associated input
    • Form check class (.form-check):works for checkboxes and radio inputs, it improves their layout and behavior
    • form check inline class(.form-check-inline): to align checkboxes and radio options on the same line
    • form check label (.form-check-label): causes label to align vertically with its CHECKBOX or RADIO inputs
    • form check input(.form-check-input): better spacing and alignment for checkboxes and radio

    Overview: Tables and Cards

    Bootstrap tables and cards are two common ways to display content on a webpage

    Html and Boostrap tables
    • in the early days html tables were used to display content
    • Now html tables are used for its initial intended use: to make tables
    • no JavaScript is necessary for an html table
    • html entities: a string that begins with an ampersand and ends with a semicolon. these entities are used to display reserved characters
    How to make HTML Tables
    1. start with {table}{table/} element and wrap it around the content
    2. within the table use {tr}{tr/} element to create table rows
    3. within the table row place as many columns as needed with {td}{td/}. Each td represents a single cell
    4. opt: you can separate the header row and the body row with{thead}{thead/}/ {tbody}{tbody/} tags
      • in this case replace the {td} tag to {th} for the header rows and the title rows
    Boostrap table
    • table class (.table):it will add bootstrap default styles
    • (.table-bordered): adds borders to the entire table and cells
    • (.table-responsive):will cause a horinzontal scroll bar to appear when the screen is too small
    Cards
    • Cards are a light-weight, flexible way to set up data for the rest of the page
    • there is no such thing as cards in html
    • by default a card has no padding or margins
    How to use Cards
    1. use (.card) to wrap a card around the data. by default the card will take up the entire width of the screen
    2. width can be determined with 'style=width:height' inside of div/section/etc.
    3. use(.card-body) to set up a padded section
    4. use (.card-header) to set up header in Cards

    More Codes

    • {dl}{dl/}: Align terms and descriptions horizontally by using our grid system’s predefined classes
      • dl: grid; dt: word to be defined;dd: definition/description
    • {cite title="Source-Title"}{cite/}: tag used when identifiying the name of a Source works
    • {blockquote}{blockquote/}: for identifying the source
    • (.blockquote-footer): styles the footer of a quote

    Overview: Images and Media

    Bootstrap Image Classes
    • Images in Bootstrap are made responsive with .img-fluid. max-width: 100%; and height: auto; are applied to the image so that it scales with the parent element.
    • use .img-thumbnail to give an image a rounded 1px border appearance.
    • use .d-flex to create a flexbox container and transform direct children elements into flex items. these containers are able to be modified with additional flex properties. Bootstrap recommends using this class to deal with a bug in internet explorer with media objects
    Media Object Classes
    • helps build complex repetitive components where some media is positioned alongside content,but doesnt wrap around said media
    How to use Media Classes
    1. use .media to wrap around the image and text
    2. use .media-body to wrap around the text that goes with its associated image
