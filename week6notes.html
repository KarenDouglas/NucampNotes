Week 6 Notes: JavaScript

Template Literals (Template Strings)

Template Literals

Template Literals

strings allowing embedded expressions.You can use multi-line strings and string interpolation features with them.

  • template literals are enclosed by the backtick, instead of quotes
  • they can contain placeholders ( they look like this: ${expression}). The expressions in the placeholders get passed to a function

Multi-line strings

Multi-line Strings

any new line characters inside of the template literal

  • instead of using the "\n" tag inside the template literal to create a new line, surround the quote in backticks and place the string on the next line in editor

Objects and Classes

Objects

Objects

a data structure for a collection of data. they can also contain properties and methods(which are variables and functions that belong to that object)

  • Objects are considered the only non-primitive data-type in JS

Arrays vs Objects

  • Arrays are referenced by their index number, while objects can be referenced by the property name
  • Arrays can only have predifined methods( like .shift, .pop, .push), while Objects can have custom methods

Object Methods

const objectName = {
property1: value1
property2: value2
methodName() {
// function code
} };

Object Constants

  • Even if you declare an object variable with the const keyword, the properties and methods can be changed.
  • similar to arrays, the variable name can not be reassigned

Method Parameters

  • You can use parameter in methods the same as in functions

Const objectName= {
property1: "propertyvalue1"
methodName ( functionParameters){
/function code here
}
//call the function like this:
objectName.methodName("argument"); };

JavaScript Classes

Classes

JavaScript Classes

templates for objects. Use them to create multiple instances of the same kind of object

  • this is completely different from the html "class='" attribute
  • classes aren't unique to JS, as they are part of the object oriented Programming (OOP) paradigm and are found in many programming languages

class ClassName { //the class name's first letter must be UPPERCASE
constructor(propertyName, propertyName2) {
this.propertyName = propertyName;
this.propertyName2= propertyName2;
}
}

to add new object (or create a new instance)

const objectName = new ClassName ( propertyValue, propertyValue2);

Class Methods

  • you can also add default methods to class:

class ClassName { //the class name's first letter must be UPPERCASE
constructor(propertyName, propertyName2) {
this.propertyName = propertyName;
this.propertyName2= propertyName2;
}
classMethodName () { // this is outside of constructor
// function code here
}

Class Inheritance

  • typically, parent classes are more general. Child classes are typically more specific

use the "extends" keyword to create child classes:

class ChildClassName extends ParentClassName { //notice that class names start with UPPERCASE letters
code goes here...;
}
}

super method

a method for child constructors to pass those properties that are inherited from the paent cass to be initialized in the constructor there.

  • You must use it with inherited properties that are defined in the paren class's constructor

this.propertyName = propertyName;
this.propertyName2= propertyName2;

Overview - Advanced Array Methods

Arrays and Methods

array

a data structure ath can be stored with a variable

methods

a function that is never called itself but attaches, using the dot notation, to a set of data like an array, or object

  • arrays follow some of the rules of an object

Advanced Array methods facts

Higher Order Function

a function that takes another function as an argument, or returns a function as a return value (ex: map,filter and reduce)

Callback Functions

A function that's passed into a higher order function as an argument, then used inside of it

iteration methods

it iterates through the array as many times as the array length, as opposed to mutating the array. (ex: map,filter, reduce

mutator methods

methods that change the original array

  • It is possible to pass a function into another function as an argument

Advanced Array Methods

Map

map array method

a method that iterates thru an array,performs a given call function on every item in the array, and returns a new array at the end.

var varName =[1,2,3,4];
function newArray = varName.map(myFunction);

// you can pass an arrow function thru map like this below;

function newArray = varName.map(variable => //code goes here) ;
// the 'variable' represents the iteration thru which to run the array.

Filter

filter

a method that takes a callback function that specifies a filterin condition, iterates through an array, and returns a new , filtered array

array.filter(function( true or false function to filter thru))

Reduce

  • Map and Filter both return a new array; Reduce returns a single value
  • while you don't need to set an initial value parameter for reduce with numbers, when working with arrays of objects, you do need toset an initial value

const variable= array.reduce((accumulatar, currentValue) => //code goes here)
//this will accumulate the array's value thru the function, returning a single value

Method Chaining

Method Chaining

when you run multiple methods in a single statement by using there return values

// example of method chaining array functions

const variableArrayName = ArrayName.method1().method2();

//example of method chaining Obects (custom methods)

const variableObjectName = objectName.method1().method2();
// this will only work when the methods have return values when working with custom methods!

  • Map,filter and reduce methods all have return values by default, but not all array methods have return values
  • methods must have return values to be chained
