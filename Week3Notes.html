Overview: Collapse and Accordion

Collaspe Component

  • helps to toggle parts of a webpage in and out of view
  • Designed to be flexible with the help of JS plugins, which can be used with data attributes
  • this can be used with BS Card component to create a type of Collapse called an Accordion

How to make Accordion

  1. Wrap main content around a div:id:"accordion".
  2. Inside that div, create a "card" div for each piece of content to collapse in accordion
  3. inside card creat a div:role:"tab"class:"card-header
  4. identify the header content and wrap "a" element around it
  5. inside the "a" element add data-toggle:"collapse" and data-target:"its assoicated link-to-id
  6. for each card except the first one, in the "a" element, add a class of "collapseD"
  7. wrap a div:class:"card-body" around relevant content usually wrapped around "p" element
  8. add "d-none" and "d-sm-block" class to the "p" element
  9. Wrap another div:class:"collapse" and id:"its related #" around the card-body div

Overview: The Nav Component

  • Set up diff kinds of navigation interfaces using either the ul or nav element and the class of "nav"
  • includes simple sets of links or more complex tabs or pills.

How to add a tabbed interface

  1. set up ul elements with li elements inside it. Label the li accordingly
  2. give ul element the class of "nav" amd "nav-tabs"
  3. add "nav-item" to all list items
  4. wrap "a" element around each name within the li elements
  5. give "a" elements the class of "nav-link"
  6. Assign link to its related name
  7. add a role of "tab" and a data-toggle of "tab" in each list item
  8. add to the class of the first "a" element "active" to make this the default open tab
  9. wrap div around the tabs associated content with the class of "tab-content"
  10. wrap div around each piece of content that will have its own tab with the role:"tabpanel", class:"tab-pane fade" id:"its associated link"
  11. add "show active" to a div class to create an open default tab

JavaScript Fundamentals

The Developer Console

  • all major browsers have a version of jS developer console.consoles are mainly used to test out small pieces of code or to interact with scripts on the current webpage.
  • The Developer Console is an ex of whats known as a R.E.P.L.- a Read Evaluate Print Loop
  • It is basically a simple interface that allows you to enter small pieces of JavaScript, immediately reads and evaluates that code, then prints out a response to it - either the value of an expression that you entered, or the return value if you entered a method or function.

Data Types, Variables, Grammar

Data types: Number, String, Boolean, Undefined (and more).

  • Number: can be an integer or have decimal oints,can be positive or negative
  • String: literal, textual data- will always have quotations around it
  • Boolean: true or false values
  • Undefined: a place in memory that has not been assigned a value.

Variable

Variables vs Literals
  • Literals do not change, they are hard coded values
  • variables is a location in the computer's memory where info can be stored and accessed by an identifier, ususally a variable name- ifo stored this way can be changed by the code
Variables: Declaration, Assignment, Initialization
Declaration
  • to declare a variable means to set aside a location in memory for it
  • declare all variables once prior to use, at the top of their scope use var/let/const to create and name a variable
Assignment
  • assign a value to the variable using an asignment operator, usually=
Initialization
  • the first time a variable is assigned a value, which can be done together with Declaration

Assignment and Arithmetic Operators

Operators

  • Arithmetic Operators( + - */ ++ --) can be used prefix and postfix. these operators actually change the value of the operand
    • ++ : Increment Operator takes a single operand, no spaces between ++ and variable name. Will increase the value of the variable by 1
    • -- :Decrement Operator takes a single operand, no spaces between an variable name. will decrement the value of the variable by 1
  • Assignment Operators (= += -= *=/=): each takes 2 operand must ve a variable. Will each perform the arithmetic operation first, then assign the result to the left operand's value

Comparison and Logical Operators

  • all comparison operators return either true or false
  • == : Equality Operator- loose equality - the data types of its operands do not need to match
  • === : Strict Equality Operators- the data types of the operands must match
  • != : inequality Operator, loose
  • !== : Strict Inequality Operator

Logical Operators

  • All logical operators retur true or false
  • All values can be converted to Boolean values
  • && : Logical And: takes 2 operands and checks if both are truthy
  • || : Logical Or : the only time this operator returns false is if all values are false
  • ! : Logical Not: it determines the opposite value of a value

Type Coercion

  • Type coercion is one way to convert a value's data type.
  • Falsy: any value that is 0,"", undefined, null or NaN
  • truthy: all other values are converted to true
