Web Development Stack

Front End/Back End/Full Stack

  • Front end=Client side (web browser)
  • Backend=Server-side
  • Full Stack=Front End + Back End

3-Tier Arhitecture Model

  • Presentation Tier=Front End
    • getting data to and from the user
  • Business Logic Tier=Back end
    • Intermediarty between presentation and data tiers
  • Data Tier=Backend
    • Storing and managing website data

Introduction to Bootstrap and Front End UI Framework

Front End Framework(library) Meaning

  • concerned with building the presentation layer of a website- content and style
  • "Bootstrap" is one of the most well known UI frameworks
  • it is mainly css based

Why Use Front End Framework

  • avoids starting from scratch: avoids issues with code,consistency and maintainability.
  • great way to learn new concept for front-end development

Responsive Web Design

  • means using responsive utilities such as view-port meta tag, media queries, and reponsive images to render a more optimized view for different devices
  • a responsive site dynamically adjusts its content based on the device size

Mobile-First?
  • most people used to browse the web on computer only, so webpages were designed for large screen
  • today, people increasingly browse the webe on various mobile devices
  • In the US More than 50% of web visits are from a mobile device

The Viewport

  • The visible area of a webpage given in width and height in pixels (not always the same as screen resolution)
  • Screen resolution also takes int account pixel density (ppi)
  • Does not include parts of the browser such as the address bar, scrollbar, bookmarks bar, etc

this viewport value initializes the webpage to be optimized for most device sizes.

Responsive design with media queries and breakpoints
  • uses @media tag to determine when a page adjusts based on screen size

Overview: Bootstrap Grid

  • Core feature of Bootstrap-a way to create a grid layout for a page to control where elements are placed
  • similar to a concept of an HTML table
  • Designed as a mobile-first, responsive layout,based mostly in css
  • Classes: Container,Row,and Col are required to make a Bootstrap Grid
  • Containers can have multiple rows. Row can have up to 12 columns. All content goes within the columns.

Boostrap Breakpoints

  • above shows the standardized sizing in Bootstrap
  1. use this to determine cell space between one and 12
  2. use this to determine when to change cell space based on device size

Bootstrap Order-* Classes

  • use order-* classes to changethe order of columns in a row

Boostrap Offset-* Classes

  • use Offset-* classes to force a Bootstrap column to the right

More Bootstrap Classes

jumbotron: used to call attention to a section of the page that shows key messages
list-unstyled: removes default list item styles

Adding Custom Styles and Bootstrap Alignment Classes

  • use Google Fonts for custom fonts in css
  • In Bootstrap (as in HTML and CSS) there are many utilities to help you with aligning images, text, columns, and other content. align-items-* classes and the text-* classes for alignment.
