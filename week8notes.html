Presentational and Container Components

Presentational VS Container Components

Presentational Component

an informal name for a component that is: concerned with how the component looks, doesn't specify how data is loade or mutated, recieves data and callback almost exclusively via props, rarely have their own state except for UI states, are written as functional components.

Container Component

an informal name for a component that: are concerned with how a component works, provide the data and behavior to other components, are often stateful, they dont render UI

Functional Components

Functional Components

  • typically written as function declarations or arrow functions
  • The first letter of component must be capitalized
  • don't use render method
  • don't use constructors or state
  • don't use "this" keyword when refering to props
  • can not use lifecycle methods

React Virtual DOM

The Browser Dom

  • DOM is Short for Document Object Model
  • The Dom act as an interface between HTML and Programming languages
  • Behind the scenes, the DOM maps the HTML elements to a node-based, tree-like representation that programming languages can understand, traverse and modify
  • it is cross-platform and language independent
  • any changes you make to the webpage's view causes the dom to be re-created- the affected DOM nodes and all their children, the entire branch
  • Whenever the DOM is recreated, the graphical view is also re-rendered, which is an expensive process/li>

Browser DOM

an API that is a set of specification for how programming languages and HTML can interface with each other

The Virtual DOM

  • The Virtual DOM technique is not unique to React
  • It is a lightweight representation of the browser DOM in memory
  • It's much faster to update the VD than BD , because it doesn't have to re-rendered the graphical display
  • in React, any changes made to the DOM are first made in the VD
  • React uses a reconciliation engine named, Fiber
  • in React we use "setState" to change the app stat instead of changing it directly: setState changes the virtual DOM
  • in React we set a unique key attribute to elements rendered as a list using "map", because this helps the reconciliation process figure out what items have changed and should be updated, and which to leave alone

reconciliation

a process that looks for changes made in the DOM, then it figures out the most optimized way to patch the browser DOm with the changes with minimal re-rendering, such as by combining updats in batches instead of multiple single changes

Overview: React Router

React Router

  • in React, you will not have multiple HTML pages, instead you can use React to build a Single Page Application(SPA), launched from a single HTML page
  • A routing library such as React Router is critical to an SPA
  • React Router is a third party library in the React ecosystem
  • RR is a collection of router, route matching and navigation components
  • it handles routing request to navigate to different views. It also handles browser history

React Router Components

<BrowserRouter>

Top-level parent component that wraps around all other React Router components. This uses HTML5 history API to control browser history

<Route>

Renders the UI for a matching path, use the "exact" attribute if you want an exact match

<Redirect>

a React Router component that Redirects to a new url

<Switch>

Groups <Route> components together

<Link>

links to a path, renders as <a>

<NavLink>

a special version of <Link>, add styles to active link

Overview: Single Page Applications

SPA

Single Page Applications

one page that locally downloads all the front-end application code for the website

Pros and Cons of SPA

  • pro: usually much faster due to fewer server requests
  • pro: easier to divide back end and front end tasks
  • pro: you can easily reuse back end code for mobile app dev
  • con: search engines have trouble indexing SPAs
  • con: its dependent on JavaScript being enabled in browser
  • con: initial download of app code can be slow
  • con: you must be careful of memory leaks as they can be more harmful
